# Lab 1a 
### Virtualisation & Linux Setup
### Ubuntu Desktop and Command Line Familiarisation

## 🔍 Objective
Set up a virtualised Linux environment, install Ubuntu, and gain familiarity with both the desktop interface and the command line.

## 🛠 Steps Taken
- Installed VirtualBox and created a new VM
- Mounted Ubuntu ISO and configured VM settings (RAM, disk, display)
- Installed Ubuntu and explored the desktop environment
- Opened the terminal and practiced basic commands:
  - `ls`, `cd` for navigation
  - `touch`, `nano`, `cat` for file creation and editing
  - `mv`, `rm` for file management
- Tested networking with `ping` and `ip a`

## Challenges
1. Challenges during VM setup  
I had trouble mounting the ISO correctly and making sure the network settings worked. Sometimes the display settings also needed adjustment.

2. Learning about virtualisation tools  
I learned that VirtualBox, VMware, and WSL all let you run Linux, but they work differently. VirtualBox is flexible, VMware is more enterprise‑focused, and WSL is built into Windows.

3. Confidence using Ubuntu  
I feel more confident now. I can use the terminal, install packages, and move around the desktop without much difficulty.

4. Industry applications of virtualised Linux  
Virtual machines are useful for testing software, running servers, practicing cybersecurity, and giving developers safe environments to work in.

5. What I’d do differently next time  
I would give the VM more RAM and disk space, take snapshots for backup, and keep better notes of each step to make troubleshooting easier.

## 💡 Reflection
Using Ubuntu through both the GUI and CLI showed me how each has unique strengths. The GUI is simple and intuitive, making tasks like browsing and installing apps through the Software Centre easy. The CLI, however, is faster and more powerful once commands are learned, especially for package management and compiling code. Each installation method has trade‑offs: SaaS is convenient but requires internet, repository installs are secure and updated automatically, binary downloads offer flexibility but risk dependency issues, and source compilation gives full control but demands technical skill. As a user I prefer GUI simplicity, but as a developer I value CLI and source compilation for deeper insight.
