# Lab 1a – Virtualisation & Linux Setup

## 🔍 Objective
Set up a virtualised Linux environment and gain hands‑on experience with Ubuntu installation.

## 🛠 Steps Taken
- Installed VirtualBox and created a new VM
- Mounted Ubuntu ISO and configured VM settings (RAM, disk, display)
- Adjusted networking (NAT vs Bridged mode)
- Installed Ubuntu and explored both GUI and CLI

## 💡 Reflection
1. Challenges during VM setup  
I had trouble mounting the ISO correctly and making sure the network settings worked. Sometimes the display settings also needed adjustment.

2. Learning about virtualisation tools  
I learned that VirtualBox, VMware, and WSL all let you run Linux, but they work differently. VirtualBox is flexible, VMware is more enterprise‑focused, and WSL is built into Windows.

3. Confidence using Ubuntu  
I feel more confident now. I can use the terminal, install packages, and move around the desktop without much difficulty.

4. Industry applications of virtualised Linux  
Virtual machines are useful for testing software, running servers, practicing cybersecurity, and giving developers safe environments to work in.

5. What I’d do differently next time  
I would give the VM more RAM and disk space, take snapshots for backup, and keep better notes of each step to make troubleshooting easier.
